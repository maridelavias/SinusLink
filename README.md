## 🩺 SinusLink — Telegram‑бот для связи стоматолога и ЛОР‑врача

Telegram‑бот, который помогает стоматологу быстро собрать структурированную анкету пациента, прикрепить снимки (КТ, фото и др.) и отправить всё в удобном виде ЛОР‑врачу для консультации.

## 📌 Краткое описание

SinusLink автоматизирует подготовку заявки на ЛОР‑консилиум:

- стоматолог заполняет короткую анкету (жалобы, анамнез, планируемая работа);
- прикрепляет снимки и файлы;
- бот формирует текстовое резюме и архив `zip` с вложениями;
- заявка отправляется в заданный Telegram‑чат ЛОР‑врача.

Проект полезен клиникам и частным специалистам, которым нужно быстро и стандартизированно передавать данные пациента для междисциплинарного обсуждения.

## 🛠 Технологии и стек

- **Язык**: Python 3.11
- **Telegram API**: `python-telegram-bot >= 21.4` (async, `ApplicationBuilder`, `ConversationHandler`, `CallbackQueryHandler`, `HTTPXRequest`)
- **Хранилище**: SQLite через `aiosqlite` (асинхронный доступ)
- **Конфигурация**: переменные окружения, `python-dotenv`
- **Логирование**: стандартный `logging` (логгер `lor-bot`)
- **Инфраструктура**:
  - Docker (образ на основе `python:3.11-slim`)
  - `docker-compose` для локального/серверного запуска
  - подготовленный конфиг и инструкция для деплоя на **Northflank** (`northflank.yaml`, `NORTHFLANK_DEPLOY.md`)
- **Архивация файлов**: стандартные модули `zipfile`, `tempfile`, работа с `BytesIO`

Основные зависимости берутся из `requirements.txt`:

- `python-telegram-bot>=21.4`
- `python-dotenv>=1.0.1`
- `aiosqlite>=0.20.0`

## 🚀 Установка и запуск

Перед запуском создайте `.env` (см. раздел про переменные окружения). **Не коммитьте реальные токены** в репозиторий.

### Запуск локально (Python)

1. **Клонировать репозиторий**:

   ```bash
   git clone https://github.com/maridelavias/SinusLink.git
   cd SinusLink
   ```

2. **Создать и активировать виртуальное окружение (опционально, но рекомендуется)**:

   ```bash
   python3 -m venv .venv
   source .venv/bin/activate  # Windows: .venv\Scripts\activate
   ```

3. **Установить зависимости**:

   ```bash
   pip install --upgrade pip
   pip install -r requirements.txt
   ```

4. **Создать файл `.env` в корне проекта**:

   ```env
   BOT_TOKEN=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   LOR_TARGET_CHAT_ID=123456789
   MAX_ZIP_MB=47          # опционально, максимум размера zip‑архива в МБ
   DATA_DIR=/data         # опционально, путь к каталогу с базой (по умолчанию .)
   ```

   - `BOT_TOKEN` — токен Telegram‑бота от `@BotFather` (обязательно).
   - `LOR_TARGET_CHAT_ID` — ID чата/канала/группы, куда бот будет отправлять заявки (обязательно).

5. **Запустить бот**:

   ```bash
   python -m app.bot
   ```

   В логах вы увидите сообщение вида:

   ```text
   Запуск long polling...
   ```

### Запуск через Docker

1. **Собрать образ и запустить контейнер**:

   ```bash
   cd SinusLink
   docker build -t sinuslink-bot .
   docker run --rm \
     --name sinuslink-bot \
     --env-file .env \
     -v $(pwd)/data:/data \
     sinuslink-bot
   ```

   Здесь `/data` внутри контейнера используется для хранения `bot.db` (SQLite).

2. **Альтернатива: `docker-compose`** — в репозитории есть готовый файл `docker-compose.yml`:

   ```bash
   docker-compose up -d --build
   ```

   По умолчанию:

   - сервис называется `lor-ent-bot`;
   - используется `.env` из корня;
   - том `botdata` монтируется в `/data` (там лежит `bot.db`).

### Деплой на Northflank

В проекте есть подробная инструкция: см. файл `NORTHFLANK_DEPLOY.md`.

Кратко:

- создаёте Deployment‑сервис в Northflank на основе этого репозитория и `Dockerfile`;
- настраиваете переменные окружения (`BOT_TOKEN`, `LOR_TARGET_CHAT_ID`, `MAX_ZIP_MB`, `DATA_DIR`);
- создаёте персистентный volume (например, `bot-data`) и монтируете его в `/data`;
- после деплоя проверяете логи — при успешном запуске увидите `Запуск long polling...`.

## ✨ Возможности

- **Главное меню**:
  - команда `/start` и кнопки клавиатуры с вариантами:
    - «🆕 Начать новую консультацию»;
    - «✍️ Заполнить профиль»;
    - «ℹ️ Мои данные».
- **Профиль стоматолога**:
  - диалог регистрации (`/fill` или соответствующая кнопка);
  - сохранение ФИО, телефона, места работы и `@username`;
  - просмотр текущих данных через `/me` или кнопку «Мои данные».
- **Заявка на консультацию**:
  - по шагам запрашиваются:
    - жалобы пациента;
    - анамнез;
    - планируемая стоматологическая работа;
    - файлы (фото и документы, можно несколько).
  - черновик автоматически сохраняется в базе после каждого шага;
  - можно продолжить незавершённую консультацию, начать заново или отменить.
- **Работа с вложениями**:
  - поддержка фото (`photo`) и документов (`document`);
  - подсчёт общего размера файлов;
  - если размер ≤ `MAX_ZIP_MB` — формируется `zip`‑архив c:
    - текстовым файлом `00_summary.txt` (plain‑text версия анкеты);
    - всеми вложениями;
  - если размер превышен или отправка архива не удалась — бот рассылает медиа‑группы с тем же описанием.
- **Отправка ЛОР‑врачу**:
  - бот формирует HTML‑описание заявки;
  - в архиве/сообщении есть удобный текст для ознакомления;
  - формируется инлайн‑кнопка «💬 Написать стоматологу» со ссылкой на профиль стоматолога (по `username` или `tg_id`).
- **Журнал заявок**:
  - таблица `consultations` хранит статус и дату;
  - команда `/list` показывает последние заявки стоматолога с инлайн‑кнопками;
  - коллбек `view_consult:<id>` раскрывает детали.
- **Надёжность и UX**:
  - обработчик ошибок `on_error` с логированием;
  - попытки gracefully пропустить ошибки Telegram API (`BadRequest`, `RetryAfter`) при настройке команд и описания бота;
  - человекочитаемые текстовые подсказки и состояния диалога.

## 🧪 Примеры использования

### Сценарий 1: первый запуск и заполнение профиля

1. Стоматолог открывает бота и отправляет `/start`.
2. Бот видит, что профиль пустой, и предлагает его заполнить.
3. Пользователь нажимает «✍️ Заполнить профиль» или команду `/fill`.
4. Бот по очереди спрашивает:
   - ФИО;
   - телефон;
   - место работы.
5. После сохранения бот показывает главное меню.

### Сценарий 2: создание и отправка консультации

1. Пользователь выбирает «🆕 Начать новую консультацию» или команду `/new`.
2. Бот по шагам запрашивает:
   - 1/4 — жалобы пациента;
   - 2/4 — анамнез;
   - 3/4 — планируемая работа;
   - 4/4 — прикрепление файлов (фото / документы).
3. Каждый вложенный файл добавляется в черновик; бот подтверждает его приём.
4. После нажатия «Готово» бот показывает превью анкеты и спрашивает подтверждение:
   - «✅ Отправить» — заявка отправляется в ЛОР‑чат;
   - «❌ Отмена» — черновик удаляется;
   - «🔄 Начать заново» — всё очищается и диалог начинается с нуля.
5. ЛОР‑врач получает либо zip‑архив (`lor_consultation.zip`), либо медиа‑группы с тем же текстом.

### Сценарий 3: продолжение черновика

1. Если стоматолог начал анкету, но не закончил, при следующем `/new` бот предлагает:
   - «▶️ Продолжить»;
   - «🔄 Начать заново».
2. При продолжении бот переходит на первый незаполненный шаг (анамнез, план или файлы).

## ❓ FAQ

- **Где хранится база данных?**  
  По умолчанию SQLite‑файл `bot.db` создаётся в директории, указанной в `DATA_DIR` (или в текущем каталоге, если переменная не задана). В Docker и на Northflank рекомендуется монтировать персистентный том в `/data` и выставлять `DATA_DIR=/data`.

- **Как изменить целевой чат для ЛОР‑врача?**  
  Установите/измените `LOR_TARGET_CHAT_ID` в переменных окружения (или `.env`). Это может быть ID личного чата, группы или канала.

- **Какой максимальный размер архива и вложений?**  
  Определяется `MAX_ZIP_MB` (по умолчанию 47 МБ). Если суммарный размер файлов больше, бот отправит медиа‑группы с тем же описанием вместо zip‑архива.

- **Поддерживает ли бот несколько инстансов?**  
  При использовании одного общего SQLite‑файла и персистентного тома рекомендуемый режим — один инстанс. Для Northflank в инструкциях явно указан 1 экземпляр при подключённом volume.

- **Можно ли использовать другой тип БД?**  
  На уровне кода сейчас заложен SQLite через `aiosqlite`. Теоретически возможно вынести слой `db.py` на другой драйвер, но это потребует модификаций.

- **Поддерживается ли Webhook‑режим вместо long polling?**  
  В текущей версии бот запускается через `app.run_polling(...)`. Для webhooks потребуется отдельный entrypoint и настройка веб‑сервера/ингресса.
