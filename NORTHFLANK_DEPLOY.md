# Развертывание бота на Northflank

## Подготовка

1. Убедитесь, что у вас есть аккаунт на [Northflank](https://northflank.com)
2. Создайте новый проект в Northflank

## Шаги развертывания

### 1. Создание сервиса

1. В панели Northflank нажмите "Create new" → "Deployment"
2. Выберите "External" как источник (будем использовать Dockerfile)
3. Подключите ваш Git репозиторий или загрузите код

### 2. Настройка переменных окружения

В разделе "Environment Variables" добавьте:

- `BOT_TOKEN` - токен вашего Telegram бота (обязательно)
- `LOR_TARGET_CHAT_ID` - ID чата для отправки заявок (обязательно)
- `MAX_ZIP_MB` - максимальный размер ZIP (по умолчанию: 47)
- `DATA_DIR` - путь для данных (по умолчанию: `/data`)

**Важно:** Используйте "Secrets" для `BOT_TOKEN` и других чувствительных данных.

### 3. Настройка персистентного хранилища

1. В настройках сервиса перейдите в раздел "Volumes"
2. Нажмите "Add volume"
3. Настройте:
   - **Name**: `bot-data`
   - **Size**: 1 GB (достаточно для SQLite базы)
   - **Container Mount Path**: `/data`
4. Сохраните

**Примечание:** При использовании персистентного тома сервис будет ограничен 1 инстансом.

### 4. Настройка ресурсов

- **Compute Plan**: Выберите подходящий план (Starter обычно достаточно)
- **Instances**: 1 (обязательно при использовании персистентного тома)

### 5. Деплой

1. Нажмите "Create service"
2. Northflank автоматически соберет Docker образ из Dockerfile
3. После успешного деплоя бот начнет работать

## Проверка работы

После деплоя проверьте логи в разделе "Logs" сервиса. Вы должны увидеть:
```
Запуск long polling...
```

## Миграция существующей базы данных

Если у вас уже есть `bot.db`:

1. Создайте backup базы данных локально
2. После деплоя подключитесь к сервису через SSH (если доступно) или используйте volume backup
3. Загрузите файл базы в `/data/bot.db`

## Обновление бота

При каждом push в подключенный репозиторий Northflank автоматически пересоберет и перезапустит сервис.

## Troubleshooting

- **Бот не отвечает**: Проверьте логи и убедитесь, что `BOT_TOKEN` установлен правильно
- **База данных не сохраняется**: Убедитесь, что volume смонтирован в `/data`
- **Ошибки при запуске**: Проверьте все обязательные переменные окружения
